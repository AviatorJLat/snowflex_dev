# SnowflexDev Requirements

## v1 Requirements

### Python Bridge
- [x] **PORT-01**: Elixir communicates with Python via Erlang Port using `{:packet, 4}` length-prefixed JSON protocol
- [x] **PORT-02**: Python worker script connects to Snowflake via `snowflake-connector-python` with `externalbrowser` SSO
- [x] **PORT-03**: Python stdout redirected to stderr on startup; protocol uses `sys.__stdout__.buffer` exclusively to prevent corruption
- [x] **PORT-04**: Python process monitors stdin for EOF and self-terminates to prevent zombie processes
- [x] **PORT-05**: Large result sets transferred in chunks to prevent memory exhaustion on both sides

### DBConnection Adapter
- [x] **DBC-01**: Implements `DBConnection` behaviour with `connect/2`, `disconnect/2`, `checkout/1`, `checkin/1`, `ping/1`, `handle_execute/4` callbacks
- [x] **DBC-02**: Query struct matches Snowflex.Query fields and behaviour
- [x] **DBC-03**: Result struct matches Snowflex.Result (columns, rows, num_rows, metadata, messages, query, query_id, request_id, sql_state)
- [x] **DBC-04**: Type decoding maps Python connector types to identical Elixir types as Snowflex (FIXEDâ†’Decimal, REALâ†’float, TIMESTAMPâ†’NaiveDateTime/DateTime, DATEâ†’Date, TIMEâ†’Time, BOOLEANâ†’boolean)
- [x] **DBC-05**: Port crash recovery â€” Python process failure triggers clean reconnect without crashing the BEAM supervision tree

### Ecto Integration
- [x] **ECTO-01**: Implements `Ecto.Adapter`, `Ecto.Adapter.Queryable`, and `Ecto.Adapter.Schema` behaviours
- [x] **ECTO-02**: Reuses Snowflex's SQL generation module (Snowflake dialect: SELECT, INSERT, UPDATE, DELETE, CTEs, window functions, QUALIFY)
- [x] **ECTO-03**: Loaders and dumpers convert between Elixir types and Snowflake column types
- [x] **ECTO-04**: Consuming app can use `Repo.all/1`, `Repo.insert/2`, `Repo.update/2`, `Repo.delete/2` identically to Snowflex

### Transport Layer
- [x] **TRANS-01**: Transport GenServer manages Port lifecycle (open, monitor, restart on crash)
- [x] **TRANS-02**: Synchronous command/response flow â€” GenServer.call blocks until Python returns result
- [x] **TRANS-03**: Configurable connection parameters: account, user, warehouse, database, schema, role, authenticator
- [x] **TRANS-04**: Connection timeout extended for SSO auth (browser popup may take 30+ seconds)

### Developer Experience
- [x] **DX-01**: `mix snowflex_dev.setup` creates Python virtualenv and installs snowflake-connector-python automatically
- [x] **DX-02**: Connection health check on startup verifies Snowflake connectivity and reports clear errors
- [x] **DX-03**: Detailed error messages for common failure modes (Python not found, venv missing, pip install failed, Snowflake unreachable)
- [x] **DX-04**: Config-only swap â€” consuming app switches between SnowflexDev and Snowflex by changing adapter in config, zero code changes
